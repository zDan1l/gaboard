KONSEP SISTEM
Penilaian Kinerja Karyawan
Berbasis Fuzzy Logic — Metode Sugeno
Pendekatan Objektif untuk Evaluasi SDM berbasis Data

---

## 1. Latar Belakang

Perusahaan sebagai jaringan restoran dengan ratusan gerai dan ribuan karyawan membutuhkan sistem penilaian kinerja yang konsisten, transparan, dan mudah dijelaskan ke seluruh level manajemen.

Pendekatan konvensional (penilaian manual atau rata-rata sederhana) berisiko subjektif dan tidak mampu menangkap kombinasi kondisi yang kompleks. Misalnya: karyawan dengan penjualan tinggi tapi kehadiran rendah — nilainya tidak bisa sekadar dirata-rata.

Fuzzy Logic metode Sugeno dipilih karena:
•	Menghasilkan output numerik langsung (crisp value) tanpa defuzzifikasi rumit.
•	Rule base IF-THEN mudah dipahami dan dikonfigurasi oleh tim HR non-teknis.
•	Mampu merepresentasikan kondisi "abu-abu" seperti performa yang hampir baik atau cukup.
•	Bisa diintegrasikan ke sistem HRIS atau payroll yang sudah berjalan.

## 4. Fuzzifikasi

Setiap variabel input dibagi menjadi 3 himpunan fuzzy menggunakan fungsi keanggotaan **trapesium**. Nilai dapat memiliki derajat keanggotaan (μ) di lebih dari satu himpunan sekaligus — inilah yang membuat fuzzy logic lebih akurat dari ambang batas kaku.

### 4.1 KPI Pencapaian (0 – 100%)

| Himpunan | Simbol | Parameter Trapezoidal (a,b,c,d) | Rentang Transisi & Puncak | Keterangan |
|---------|--------|-------------------------------|------------------------|------------|
| Rendah | L | (0, 0, 40, 60) | Naik: 0→40, Puncak: 40→60, Turun: 60→60 | Target tidak tercapai atau jauh di bawah |
| Sedang | M | (50, 60, 80, 85) | Naik: 50→60, Puncak: 60→80, Turun: 80→85 | Target tercapai sebagian |
| Tinggi | H | (78, 85, 100, 100) | Naik: 78→85, Puncak: 85→100 | Target tercapai penuh atau melampaui |

### 4.2 Tingkat Kehadiran (0 – 100%)

| Himpunan | Simbol | Parameter Trapezoidal (a,b,c,d) | Rentang Transisi & Puncak | Keterangan |
|---------|--------|-------------------------------|------------------------|------------|
| Rendah | L | (0, 0, 60, 80) | Naik: 0→60, Puncak: 60→80, Turun: 80→80 | Sering absen, izin berlebihan |
| Sedang | M | (75, 80, 90, 95) | Naik: 75→80, Puncak: 80→90, Turun: 90→95 | Kehadiran umumnya terpenuhi |
| Tinggi | H | (90, 95, 100, 100) | Naik: 90→95, Puncak: 95→100 | Hampir selalu hadir tepat waktu |

### 4.3 Kepuasan Pelanggan (1 – 10)

| Himpunan | Simbol | Parameter Trapezoidal (a,b,c,d) | Rentang Transisi & Puncak | Keterangan |
|---------|--------|-------------------------------|------------------------|------------|
| Rendah | L | (1, 1, 4, 5.5) | Naik: 1→4, Puncak: 4→5.5, Turun: 5.5→5.5 | Banyak keluhan, pelayanan di bawah standar |
| Sedang | M | (4.5, 5.5, 7.5, 8) | Naik: 4.5→5.5, Puncak: 5.5→7.5, Turun: 7.5→8 | Pelayanan cukup, ada ruang perbaikan |
| Tinggi | H | (7, 8, 10, 10) | Naik: 7→8, Puncak: 8→10 | Pelanggan puas, jarang ada komplain |

## 5. Fungsi Keanggotaan Trapezium

### 5.1 Matematika Trapezoidal

**μ(x) = 0** jika x ≤ a atau x ≥ d (di luar range)
**μ(x) = (x-a)/(b-a)** jika a < x < b (naik)
**μ(x) = 1** jika b ≤ x ≤ c (puncak penuh)
**μ(x) = (d-x)/(d-c)** jika c < x < d (turun)

Fungsi ini menghasilkan nilai derajat keanggotaan antara 0 dan 1.

### 5.2 Parameter (a, b, c, d)

Setiap himpunan fuzzy didefinisikan dengan 4 parameter:
- **a**: Batas bawah (di bawah ini μ=0)
- **b**: Mulai naik ke μ=1
- **c**: Mulai turun dari μ=1
- **d**: Batas atas (di atas ini μ=0)

**Contoh: KPI "Sedang" dengan (50, 60, 80, 85)**
- x=55: μ = (55-50)/(60-50) = 0.5 (sedang naik)
- x=70: μ = 1.0 (puncak penuh)
- x=82: μ = (85-82)/(85-80) = 0.6 (sedang turun)
- x=45: μ = 0.0 (di luar range)
- x=90: μ = 0.0 (di luar range)

### 5.3 Cara Membaca Parameter dengan Benar

⚠️ **PENTING:** Jangan salah membaca parameter!

**Salah:** KPI Tinggi = 78-100 (asumsi naik dari 78 sampai 100)
**Benar:** KPI Tinggi = (78, 85, 100, 100)
  - Artinya: naik dari 78→85, puncak penuh 85→100, turun dari 100→100 (tidak turun)
  - Jadi μ=1.0 hanya pada range 85-100, bukan 78-100

**Kenapa begitu?** Karena fuzzy logic mewakili ketidakpastian dan area transisi:
- **78-84**: Area "dikit tinggi" (μ naik dari 0 ke 1)
- **85-100**: Area "pasti tinggi" (μ = 1.0)
- Jika ditulis "78-100", orang akan sangka μ=1.0 di seluruh range ini (padahal tidak)

## 7. Inferensi & Defuzzifikasi

### 7.1 Inferensi (Operator MIN)

Untuk setiap rule yang aktif, derajat aktivasi dihitung dengan operator AND → MIN:

**μ_rule = MIN(μ_KPI, μ_Kehadiran, μ_Kepuasan)**

### 7.2 Defuzzifikasi (Weighted Average — khas Sugeno)

**z* = Σ(μᵢ × zᵢ) / Σ(μᵢ)**

Hasilnya adalah skor kinerja crisp dalam rentang **0.00 – 1.00**.

### 7.3 Contoh Perhitungan

**Data karyawan:** KPI = 78%, Kehadiran = 88%, Kepuasan Pelanggan = 7.5

| Langkah | Proses | Hasil |
|--------|--------|-------|
| Fuzzifikasi | KPI=78% → μ-Rendah=0.0, μ-Sedang=1.0, μ-Tinggi=0.0 | Dominan: Sedang (μ=1.0) |
| Fuzzifikasi | Hadir=88% → μ-Rendah=0.0, μ-Sedang=1.0, μ-Tinggi=0.0 | Dominan: Sedang (μ=1.0) |
| Fuzzifikasi | Kepuasan=7.5 → μ-Rendah=0.0, μ-Sedang=0.5, μ-Tinggi=0.5 | Split: Sedang (μ=0.5), Tinggi (μ=0.5) |
| Inferensi R7 | MIN(1.0, 1.0, 1.0) × z=0.55 | μ_R7 = 1.0 (KPI Sedang ∧ Hadir Sedang ∧ Puas Sedang) |
| Inferensi R9 | MIN(1.0, 1.0, 0.0) × z=0.44 | μ_R9 = 0.0 (TIDAK AKTIF - Satisfaction Rendah=0) |
| Inferensi R6 | MIN(0.0, 1.0, 1.0) × z=0.68 | μ_R6 = 0.0 (TIDAK AKTIF - KPI Tinggi=0) |
| Inferensi R8 | MIN(1.0, 0.0, 0.0) × z=0.50 | μ_R8 = 0.0 (TIDAK AKTIF - Attendance Tinggi=0) |
| Inferensi R4 | MIN(1.0, 0.0, 0.5) × z=0.78 | μ_R4 = 0.0 (TIDAK AKTIF - Attendance Tinggi=0) |
| Defuzzifikasi | (1.0×0.55 + 0.5×0.65) / (1.0 + 0.5) | z* = 0.58 |
| **Hasil Akhir** | Skor = 0.58 → Kategori: Cukup | ✓ Rekomendasi: Program Pelatihan & Mentoring |

**Penjelasan Active Rules:**
- **Rule R7 aktif dengan strength 1.0**: KPI Sedang (1.0) AND Attendance Sedang (1.0) AND Satisfaction Sedang (1.0) → z=0.55
- **Rule (Sedang, Sedang, Tinggi) aktif dengan strength 0.5**: KPI Sedang (1.0) AND Attendance Sedang (1.0) AND Satisfaction Tinggi (0.5) → z=0.65
- Rule lain tidak aktif karena salah satu membership = 0

**Skor Akhir: 0.58 (Cukup)**

### 10.5 Rule Logic yang Diimplementasikan

**Implementasi menggunakan PHP Native di `FuzzyLogicService.php`:**

```php
// Fuzzification - Fungsi Keanggotaan Trapezium dengan parameter (a,b,c,d)
protected function fuzzifyKPI(float $value): array {
    return [
        'rendah' => $this->trapezoidalMembership($value, 0, 0, 40, 60),
        'sedang' => $this->trapezoidalMembership($value, 50, 60, 80, 85),
        'tinggi' => $this->trapezoidalMembership($value, 78, 85, 100, 100),
    ];
}

protected function fuzzifyAttendance(float $value): array {
    return [
        'rendah' => $this->trapezoidalMembership($value, 0, 0, 60, 80),
        'sedang' => $this->trapezoidalMembership($value, 75, 80, 90, 95),
        'tinggi' => $this->trapezoidalMembership($value, 90, 95, 100, 100),
    ];
}

protected function fuzzifySatisfaction(float $value): array {
    return [
        'rendah' => $this->trapezoidalMembership($value, 1, 1, 4, 5.5),
        'sedang' => $this->trapezoidalMembership($value, 4.5, 5.5, 7.5, 8),
        'tinggi' => $this->trapezoidalMembership($value, 7, 8, 10, 10),
    ];
}

// Rule Application - Operator MIN untuk AND
$strength = min($kpi['tinggi'], $attendance['tinggi'], $satisfaction['tinggi']);

// Defuzzification - Weighted Average (khas Sugeno)
$z* = Σ(μᵢ × zᵢ) / Σ(μᵢ)
```

**Total Rule: 27 rules** mencakup semua kombinasi 3×3×3:
- 12 rule utama (dengan z value 0.92 - 0.08) untuk performa ekstrem
- 15 rule tambahan (untuk kelengkapan kombinasi yang mungkin)

## 11. Logic Pengambilan Data & Penanganan Data Tidak Lengkap

### 11.1 Prinsip Dasar Penilaian

Sistem menggunakan prinsip **"Data-Driven & Fair"** dengan aturan:

1. **SKIP jika tidak lengkap** - Karyawan WAJIB memiliki minimal data untuk ketiga variabel
2. **Hitung yang ada** - Perhitungan berdasarkan data yang tersedia (5 dari 10 = hitung 5)
3. **No Perfect Score for No Data** - Tidak ada nilai sempurna untuk data yang kosong

### 11.2 Persyaratan Minimal Data

Karyawan HANYA dapat dinilai jika memiliki:

| Variabel | Minimal Data | Contoh |
|----------|--------------|--------|
| **KPI** | Minimal 1 KPI target | Ada 10 target, 5 ada report → Dinilai dari 5 yang ada |
| **Kehadiran** | Minimal 1 data absensi | Ada 1 hari kerja → Dinilai dari 1 hari itu |
| **Kepuasan** | Minimal 1 survei pelanggan | Ada 1 survei → Dinilai dari 1 survei itu |

**Jika salah satu variabel KOSONG → Karyawan DI-LEWATI (SKIP)**

### 11.3 Aturan Perhitungan per Variabel

#### **KPI Score (0-100%)**
- **Logic:** Hanya hitung target yang ADA report-nya
- **No target:** 0% (bukan 100%)
- **5 dari 10 target ada report:** Rata-ratakan 5 yang ada
- **10 target, 0 report:** 0% (bukan 100%)

#### **Attendance Rate (0-100%)**
- **Logic:** (Hadir + Terlambat) / Total Hari Kerja × 100%
- **No data:** 0% (bukan 100%)
- **1 hari kerja, hadir:** 100%
- **10 hari kerja, 8 hadir:** 80%

#### **Customer Satisfaction (1-10)**
- **Logic:** Rata-rata skor survei yang ada
- **No data:** 1.0 (minimum, bukan 10.0)
- **5 survei:** Rata-rata 5 survei
- **1 survei skor 8:** 8.0

### 11.4 Feedback Batch Generate

Saat HR klik "Generate Semua", sistem memberikan feedback detail:

```

  • Siti Rahayu: Tidak ada data kehadiran
  • Ahmad Fauzi: Tidak ada data kepuasan pelanggan
  • Dewi Lestari: Data tidak lengkap: Tidak ada data KPI (target/report), Tidak ada data kepuasan pelanggan
  • ...
```

**Keuntungan Feedback Detail:**
- HR tahu persis k mana yang perlu dilengkapi
- Tidak ada tebakan "kenapa karyawan X tidak dinilai"
- Accountability jelas untuk setiap missing data

### 11.5 Fairness & Professionalism

| Aspek | Pendekatan Lama | Pendekatan Baru (Current) |
|-------|-----------------|----------------------------|
| **No data KPI** | 100% (perfect) | 0% (minimum) |
| **No data absensi** | 100% (perfect) | 0% (minimum) |
| **No data survei** | 10/10 (perfect) | 1.0 (minimum) |
| **5 dari 10 ada report** | 100% (asumsi all perfect) | Rata-rata 5 yang ada (real) |
| **Karyawan tanpa data** | Dinilai perfect | DI-SKIP (fair) |

**Kenapa lebih fair?**
- Karyawan rajin dengan data lengkap tidak "dirugikan" oleh karyawan tanpa data
- Manager tidak bisa "main angka" dengan sengaja tidak input data
- Skor refleks performa NYATA, bukan asumsi
- HR tahu persis data apa yang missing (perbaikan jelas)
